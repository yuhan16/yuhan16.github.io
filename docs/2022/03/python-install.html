Python Install

Download source files and do configure.

./configure --prefix=/path/to/python/Python-xxx --enable-optimizations		# use Python-xxx for naming convention

Note: Use ./configure --help less to check detailed configurations.

Then use make to build Python and install the binary after the building:

$ make -j$(nproc)       # use all cpus for compile
$ sudo make install     # add sudo for admin install

Then add to path in .bashrc and source it:

$ export PATH=$PATH:/opt/Python-xxx
$ source .bashrc

Optional

Install the package virtualenv:

$ sudo /path/to/python/Python-xxx/bin/pip3 install virtualenv

The package will be installed in opt directory if using sudo instead of .local directory.

Note: venv module is contained with official Python distributions. It contains partial features of the python package virtualenv.

Note: in /opt/Python-xxx/bin directory, there are python and pip executables. So we should treat them as two independent executables. So we can directly use pip install xxx instead of python -m pip install xxx. The latter will report an error: No module named pip3.

Note: when use sudo in front of /opt/Python-xxx/bin/python and pip, the packages/updates willl be installed in opt instead of .local.

Note that virtualenv is a package of python. Unlike executable pip3, we create a new virtual environment using:

python3.x -m venv myvenv    # create myvenv environment in the current directory.
